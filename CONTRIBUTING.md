# Contributing Guide
Thank you for contributing!

## Branch Naming
- develop/short-description
- bugfix/short-description
- hotfix/short-description

Examples:
- develop/add-object
- bugfix/collision-detection
- hotfix/fix-playerjump

## Pull Request Expectations
  - Clear description
  - Screenshots (if UI changes)
  - Linked issue number
  - Tests added
    
## Before opening a PR:
- Ensure the project runs in Godot without errors
- No new warnings in the Output panel
- Test the feature manually
- Add a clear description of changes
- Include screenshots or screen recordings for UI/visual changes

## Review Expectations
- At least 1 team member must approve
- Address requested changes before merging
- Be respectful and constructive in feedback

## Running the Project Locally

1. Install the correct version of Godot (see README).
2. Clone the repository.
3. Open the project in Godot.
4. Press F5 to run the game.

## How to run checks locally
  How to run tests
    - Run the game in Debug mode on GoDot Engine
  How to run linter
    - Debugger will identify the potential errors, bugs or inefficiency in source code
  How to build the project
    - Build the game in GoDot engine
