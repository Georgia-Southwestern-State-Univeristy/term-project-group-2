Branch naming:
  develop  /*New development to be deployed under this branch */
  bugfix  /*Bugs are fixed under this branch */
  hotfix  /*Production failure fix are through hotfix */

PR (Pull Request) expectations
  - Clear description
  - Screenshots (if UI changes)
  - Linked issue number
  - Tests added

Review expectations
  - Atleast one team member to peer review

How to run checks locally
  How to run tests
    - Run the game in Debug mode on GoDot Engine
  How to run linter
    - Debugger will identify the potential errors, bugs or inefficiency in source code
  How to build the project
    - Build the game in GoDot engine
